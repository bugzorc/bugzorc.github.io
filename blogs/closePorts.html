CLOSE PROCESS RUNNING BASED ON PORT

A program couldn't start because port 3020 was already in use by another process. In this blog, we'll learn how to identify the process using a specific port.

Solution:

Open CMD
press WIN + R then type ENTER

Commands to type
Now we going find out the PID of the process using the port.
type netstat -ano | findstr : 3020

Bingo! That process is lacking on pid 1908. Now we gonna use our enchanted command of sharpness 32k to send the process into the tech abyss.
type taskkill /F /PID 1908

RIP process 1908, but we gotta be for sure dude is gone for real.
