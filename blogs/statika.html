Why I use my custom blog system

When using github pages we can't code in any server running language or framework, which makes one to be very limited. I have seen many other ways to generate blogs like jenkyll, though I love it am dreadfully in love with this site's theme plus am not gonna waste time modify jenkyll's source code to generate stuff like code snippets. Markdown? Oh, I'd check on marked library, tho I was about to reach my goal, it does all the feature that suites plus it doesn't have alot of features... I give it the name STATIKA, as in good for static pages.

How pages are generated

A javascript file reads the url which is like example.com/blog.html?file=statika.html copies the html from the file named blog.html from the blogs folder, it into the div with the class blogContent, then it's automatically rendered by the browser, css to modify the blog is in blog.css.

TIP:to write blogs fast, use VSCODE or any IDE that generates html with shortcuts, of example if I want to add <span class="block"></span>, I would just type span.block. then press Enter.

Statika also comes a with print button, which gets content from the div with the ClassName blogContent. Source code for the print button is in print.js, you can make anything clickable just add onclick="printBlog()", also make source to include the src path in ur header.
